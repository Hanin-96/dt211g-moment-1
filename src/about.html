Arbetsprocess

Utveckling & publicering

Skapa en undersida i webbplatsen där du i detalj beskriver hur din arbetsprocess för utveckling och publicering fungerar. Beskriv vad som är automatiserat och vilka verktyg du använt för detta.


Under arbetets gång har jag har jag skapat ett nytt npm projekt i terminalen för specifik arbetsmapp. Startade upp med kommandot "npm init". Därefter har parcel installerats i VSC via terminalen med kommando "npm install parcel --save-dev". Med save-dev talar om att parcel endast används under utveckling och inte publicering. En ny package.json fil skapas och i den läggs script till som talar om vad som ska startas och byggas. Vissa mapper som dyker upp innehåller många filer som inte behöver versionshanteras, därför läggs dem in i en .gitignore fil som ser till att dem inte versionshanteras när arbetet kopplas till Github

Github kopplas med arbetet för att kunna commita uppdateringar av innehåll. Under arbetets gång skapas en ny develop branch för att lägga till nytt innehåll på webbsidorna som senare merges till main branch och pushas till webbsidorna

För att kunna se ändringar som görs, skrivs kommandot "npm run start" som startar upp en lokal server för att se ändringar som görs.
